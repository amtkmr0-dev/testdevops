Jenkins CI/CD Pipeline

Complete Docker to AWS ECR Deployment

End-to-End Automated Deployment Pipeline

What This Pipeline Does

  • Automates complete deployment workflow from code to production
  • Builds Docker images from your application code
  • Pushes images to AWS ECR (Elastic Container Registry)
  • Deploys containerized app to EC2 instance automatically

Pipeline Architecture

Declarative Pipeline

Uses Jenkins DSL syntax with predefined structure

5 Core Stages

Sequential execution with environment configuration

Agent Configuration

agent any

  • Allows pipeline to run on any available Jenkins agent
  • Provides flexibility for distributed builds
  • Jenkins selects first available executor

Environment Variables (Part 1)

  • AWS_REGION: ap-south-1 (Mumbai region)
  • ECR_REGISTRY: Your AWS ECR endpoint URL
  • ECR_REPO: jenkins/repo (repository name)
  • IMAGE_TAG: Uses Jenkins BUILD_NUMBER for versioning

Environment Variables (Part 2)

  • APP_EC2_HOST: Target EC2 IP (3.108.217.189)
  • CONTAINER_NAME: student-app (running container name)
  • APP_PORT: Port 80 for web traffic
  • ECR_IMAGE & ECR_LATEST: Full image URIs with tags

Stage 1: Checkout Code

checkout scm

  • Pulls latest code from Source Control Management (Git)
  • SCM configuration defined in Jenkins job settings
  • Ensures build uses most recent codebase

Stage 2: Build Docker Image

  • Builds Docker image from Dockerfile in workspace
  • Tags with BUILD_NUMBER for version tracking
  • Creates "latest" tag for easy reference
  • Both tags point to same image (different identifiers)

Docker Build Explained

docker build -t jenkins/repo:42 .

  • -t flag specifies tag name with version
  • Dot (.) means use current directory Dockerfile
  • Creates local image ready for tagging

Stage 3: Login to AWS ECR

  • Authenticates Docker client with AWS ECR
  • Uses AWS CLI to get temporary password token
  • Pipes password directly to docker login command
  • Required before pushing images to ECR

ECR Authentication Process

aws ecr get-login-password --region ap-south-1 | docker login --username AWS --password-stdin <ECR_URL>

  • AWS CLI retrieves 12-hour token from ECR
  • Username is always "AWS" for ECR
  • Password passed via stdin for security

Stage 4: Tag & Push to ECR

  • Re-tags local images with full ECR registry path
  • Pushes both versioned and latest tags
  • Stores images in AWS cloud registry
  • Makes images available for deployment

Why Double Tagging?

Versioned Tag

Immutable build history (42, 43, 44...)

Latest Tag

Always points to most recent successful build

Stage 5: Deploy to EC2

  • Uses SSH to connect to target EC2 instance
  • Authenticates with ECR from EC2 instance
  • Pulls latest image from ECR
  • Stops old container and starts new one

SSH Agent Configuration

sshagent(credentials: ['app-ec2-ssh'])

  • Jenkins SSH Agent plugin manages SSH keys
  • Credential ID references stored private key
  • Enables secure passwordless authentication

Remote Deployment Steps (Part 1)

  • StrictHostKeyChecking=no: Bypasses SSH fingerprint verification
  • ubuntu@IP: Connects as ubuntu user to EC2
  • ECR Login: Authenticates Docker on EC2 with ECR

Remote Deployment Steps (Part 2)

  • Docker Pull: Downloads latest image from ECR to EC2
  • Docker rm -f: Force stops and removes old container
  • || true: Prevents failure if container doesn't exist

Starting New Container

docker run -d --name student-app -p 80:80 <ECR_IMAGE>

  • -d runs container in detached mode (background)
  • --name assigns container identifier
  • -p 80:80 maps host port 80 to container port 80

Post-Build Cleanup

docker system prune -f

  • Runs after pipeline regardless of success/failure
  • Removes unused Docker objects (images, containers)
  • Prevents disk space exhaustion on Jenkins agent

Complete Pipeline Flow

1. Checkout

2. Build

3. ECR Login

4. Push

5. Deploy

Prerequisites for This Pipeline

  • Jenkins with Docker and AWS CLI installed
  • AWS IAM role with ECR permissions attached to Jenkins EC2
  • ECR repository created (jenkins/repo)
  • SSH key pair configured in Jenkins credentials
  • Target EC2 with Docker and AWS CLI installed

Required AWS Permissions

Jenkins EC2

ecr:GetAuthorizationToken, ecr:PutImage, ecr:BatchCheckLayerAvailability

App EC2

ecr:GetAuthorizationToken, ecr:BatchGetImage, ecr:GetDownloadUrlForLayer

Security Considerations

  • SSH credentials stored securely in Jenkins
  • IAM roles used instead of hardcoded AWS keys
  • ECR authentication tokens expire after 12 hours
  • StrictHostKeyChecking disabled only for automation

Benefits of This Pipeline

  • Fully automated deployment reduces manual errors
  • Version tracking with BUILD_NUMBER tags
  • Zero-downtime deployment with container replacement
  • Centralized image registry with AWS ECR

Potential Enhancements

  • Add testing stage before deployment
  • Implement rollback mechanism for failed deployments
  • Add Slack/email notifications for build status
  • Use blue-green deployment for zero downtime

Troubleshooting Common Issues (Part 1)

  • ECR Login Fails: Check IAM role permissions on Jenkins EC2
  • SSH Connection Fails: Verify security group allows SSH from Jenkins
  • Docker Build Fails: Check Dockerfile syntax and dependencies

Troubleshooting Common Issues (Part 2)

  • Push Timeout: Network connectivity issue to ECR endpoints
  • Container Won't Start: Check app logs with docker logs
  • Port Already in Use: Old container wasn't removed properly

Execution Workflow Summary

  • Code commit triggers Jenkins pipeline
  • Docker image built and versioned automatically
  • Image pushed to secure AWS ECR registry
  • EC2 instance pulls and deploys new container
  • Old container replaced seamlessly
